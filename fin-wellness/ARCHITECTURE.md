# 财务健康助手 (Fin-Wellness) — 产品 & 技术架构

> 非记账工具。是一个陪伴式 AI 财务助手，帮用户改善财务状态，而不只是记录。

## 一、产品核心逻辑

### 痛点 → 解决方案映射

| # | 痛点 | 根因 | 解决方案 |
|---|------|------|----------|
| 1 | 懒得记 | 记账产品功能臃肿，摩擦力大 | **懒人模式**：CSV/截图导入，基金净值自动更新，最少手动输入 |
| 2 | 不敢看 | 财务焦虑 — 记完更焦虑 | **情绪感知对话**：不评判、不说教，正常化消费行为，引导而非恐吓 |
| 3 | 看不懂 | 消费和投资数据割裂 | **净资产看板**：一个数字看全貌（现金+投资+负债=净资产） |
| 4 | 改不动 | 只有报表没有行动路径 | **微行动循环**：发现→解释→建议→承诺→跟进，一次只改一件事 |
| 5 | AI期待高 | 通用AI不懂个人财务上下文 | **分层记忆**：记住你的习惯、偏好、目标，千人千面 |
| 6 | 数据安全 | 财务数据极敏感 | **Local-first + 选择性加密上云** |

### 核心循环：Record → Discover → Explain → Suggest → Commit → Follow-up

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   ① Record (低摩擦记录)                                      │
│   · CSV 批量导入银行/支付宝/微信账单                           │
│   · 截图 OCR 识别                                            │
│   · 基金净值自动拉取（天天基金/蛋卷 API）                      │
│   · 手动记一笔（最简 UI：金额+分类+备注）                     │
│                                                             │
│   ② Discover (AI 发现异常)                                   │
│   · "本月外卖比常态高 38%"                                    │
│   · "连续 3 个月净资产没变化"                                 │
│   · "你的基金持仓集中度过高（80% 在同一行业）"                 │
│                                                             │
│   ③ Explain (AI 解释原因)                                    │
│   · "外卖增加主要集中在周二/周四，都是加班日"                  │
│   · "净资产不变是因为消费增速=收入增速"                        │
│   · 不评判，只陈述事实                                        │
│                                                             │
│   ④ Suggest (AI 给微建议)                                    │
│   · 只给 1 个可执行的小行动，不给 10 条鸡汤                   │
│   · "试试周二/周四带饭？每月能省 ¥400"                        │
│   · "把 XX 基金的 20% 转到 YY 行业，分散风险"                 │
│                                                             │
│   ⑤ Commit (用户承诺)                                        │
│   · 用户点击 [接受] 或 [跳过] 或 [改一下]                     │
│   · 接受后变成一个"目标"，有截止时间                           │
│                                                             │
│   ⑥ Follow-up (AI 跟进)                                     │
│   · 下周只回看这一个目标                                      │
│   · "上周带饭 2 次，外卖支出降了 22%，继续？"                 │
│   · 完成 → 庆祝 + 新目标 | 没完成 → 不评判 + 调整建议        │
│                                                             │
└────────────────────────────── 🔄 循环 ──────────────────────┘
```

## 二、技术架构

### 设计原则
1. **单编排器 + 工具调用**：不搞多 Agent，一个主助手理解意图、调用工具、生成回复
2. **AI 不瞎猜财务事实**：所有数字来自数据层，AI 只负责解释和建议
3. **Local-first**：SQLite 本地存储，敏感数据不出设备；云端只存加密后的备份
4. **MVP 极简**：先跑通核心循环，再加花哨功能

## 三、MVP 范围（4 周）

### Week 1: 能记录
- SQLite schema + 数据层
- CSV 导入（支付宝/微信/银行通用格式）
- 智能分类（规则 + Claude 兜底）
- CLI: `fw import alipay.csv`

### Week 2: 能看懂
- 净资产计算（现金 + 投资 - 负债）
- 月度消费汇总
- CLI: `fw status` → 一屏看全貌

### Week 3: 能对话
- AI 编排器 + 工具调用
- 记忆层（短期 + 中期）
- 异常检测 + 解释
- CLI: `fw chat` → "本月花了多少？" / "哪里花多了？"

### Week 4: 能改变
- 微行动循环（Suggest → Commit → Follow-up）
- 周报生成
- 目标跟进
- CLI: `fw review` → 周复盘

## 四、情绪设计原则（Prompt Engineering）

AI 助手的语气规则（写进 system prompt）：

1. **永远不说"你花太多了"** → 说"这个月外卖比平时多了一些"
2. **永远不用"应该"** → 用"你可以试试"、"一个想法是"
3. **承认花钱是正常的** → "加班点外卖完全合理，只是看看有没有更划算的方式"
4. **一次只建议一件事** → 信息过载 = 什么都不做
5. **完成目标时要庆祝** → "带饭坚持了 3 周，省了 ¥600，你太厉害了"
6. **没完成时不评判** → "上周没带饭也没关系，要不要调整一下目标？"
7. **用户说"不想看"时尊重** → "好的，等你想看的时候随时叫我"
