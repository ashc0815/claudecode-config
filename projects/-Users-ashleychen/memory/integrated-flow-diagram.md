## 五、关键设计决策总结

```
Q: 为什么 OpenClaw 在 Concur 前面？
A: 员工先过我们 → 问题在进入审批流之前就被拦截
   → Concur 里只有合规的报销单 → 经理审批效率提升

Q: 为什么还要事后审计？事前不是已经查过了？
A: 三个原因:
   1. Concur 里经理可能改金额 → 改后金额需要重新核对
   2. 跨报销单的模式 (拆单/集中) 只有攒够数据才能发现
   3. 统计异常需要时间维度 → 某员工这个月报销突然涨200%

Q: 数据流方向是什么？
A: 上行: 员工凭证 → OpenClaw审核 → 写入Concur
   下行: Concur审批数据 → 拉回OpenClaw → 交叉核对
   双向同步，OpenClaw 同时是入口和质检站
```
